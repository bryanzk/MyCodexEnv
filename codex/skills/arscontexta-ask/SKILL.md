# Ars Contexta Ask

## 概述

用于回答“系统为什么这样设计”“当前图谱告诉我什么”这类问题，强调证据链（evidence chain）而非主观猜测。

## 快速流程

1. 解析问题类型：方法论、状态解释、结构疑问。
2. 在现有笔记与 ops 信息中定位依据。
3. 给出答案并附最小证据引用（文件路径/关键字段）。
4. 若证据不足，明确缺口并给补数建议。

## 输出契约

- 直接回答
- 证据来源（路径级）
- 不确定性说明（如有）

## 常见错误

- 把“建议”当成“事实”。
- 无证据引用直接下结论。
