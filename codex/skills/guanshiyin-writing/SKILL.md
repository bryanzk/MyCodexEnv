# 观世音菩萨写作法

当用户明确要求用“观世音菩萨写作法”“观世音四维”“刘文典该方法”，或点名按“观 / 世 / 音 / 菩萨”四项写作、改稿、评文时，使用本 skill。

## 边界与邻近 skill

- 普通“去 AI 味”、降低机器腔、让中文更像人写，使用 `humanizer`，不要加载本 skill。
- UX copy、提示文案、表达澄清、需求措辞优化，使用 `clarify`，不要加载本 skill。
- 交付前的成稿润色、压缩、排版和最后 polish，使用 `polish`，不要加载本 skill。
- 普通摘要、信息提取、结构化整理、事实归纳，不加载本 skill，除非用户明确要求按四维评审或写作。

## 核心原则

- **观：观察生活**。先补足真实生活细节：人物、场景、动作、物件、时间、气味、声音、矛盾。避免空泛概念先行。
- **世：世事洞明**。透过现象提炼判断：利益结构、人情逻辑、时代背景、制度约束、人的两难。不要只做情绪宣泄。
- **音：音韵铿锵**。让语言有节奏：长短句交错，关键处用短句落槌；适度使用排比、对偶、回环，但不堆砌辞藻。
- **菩萨：善良悲悯**。立意要有对人的体恤。批评可以锋利，但不要刻薄；揭示问题时保留人性复杂和向善的可能。

## 工作流程

1. **明确文体与对象**
   - 判断任务是新写、改写、润色、评论、演讲稿、短视频文案还是书信。
   - 保留用户给定的主题、立场、事实和目标读者；缺少关键信息时先基于现有材料完成，不虚构具体事实。

2. **搭建四维素材**
   - 观：列出可写入正文的具体细节。
   - 世：提炼一到三个核心洞察，优先选择最有解释力的那个。
   - 音：确定语言节奏，是沉静、激昂、讽刺、温厚还是苍凉。
   - 菩萨：确定文章最终照亮什么人、守护什么价值。

3. **组织文章**
   - 开头从具体场景、人物动作或一句有分量的判断进入。
   - 中段用细节承载观点，用观点统摄细节。
   - 结尾回到人心、人情或公共价值，避免口号式升华。

4. **打磨语言**
   - 把抽象词换成可感知的画面。
   - 把松散句改成有停顿、有重音的句群。
   - 删除空泛赞美、套话、过度煽情和没有事实支撑的大判断。
   - 保持悲悯，不把复杂的人写成单薄的标签。

## 输出要求

- 默认直接给出成稿或改稿，不额外解释方法。
- 如果用户要求评审，按“观 / 世 / 音 / 菩萨”四项给出简洁诊断和修改建议。
- 如果用户要求展示过程，可以先给“四维提纲”，再给成稿。

## 质量检查

交付前自检：

- 观：是否有可触摸的生活细节；常见失败是只有抽象概念、形容词和价值判断。
- 世：是否有独立洞察；常见失败是复述常识、只喊立场或把复杂问题写成单因果。
- 音：是否读起来有节奏；常见失败是平铺直叙、句长单一、排比堆砌或辞藻压过意思。
- 菩萨：是否有善意和悲悯；常见失败是廉价煽情、道德审判或把人写成标签。

评审评分可用 10 分制：

- 观、世、音、菩萨各 0-2 分。
- 整体完成度 0-2 分，评估结构完整、事实克制和读者适配。
- 低于 8 分时，先指出最影响成稿质量的一项，再给最小修改建议。

详细路由与评估用例见 `references/eval-matrix.md`。
