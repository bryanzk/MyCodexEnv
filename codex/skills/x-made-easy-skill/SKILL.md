# X Made Easy（"轻松学"系列书写作技能）

把用户指定的任意主题（奥数、某软件、某理论、某门手艺……）写成一本仿汤普森《Calculus Made Easy》风格的教学书：完整骨架、分章 Markdown、可合成 PDF。适合把一门看似吓人的知识，整理成一本可逐章推进的"轻松学"小书。

## 铁律（最重要，先读这三条）

1. **是"驱除恐惧"，不是写教科书。** 全书第一目的是把读者从对术语、符号、"我没天赋"的恐惧里解救出来。先除恐，再讲规矩。
2. **事实核查，绝不捏造。** 例题与事实必须真实、正确，凡数值都要验算；真实经典内容标"经典/民间/经典题型"，自己编的标"自编"——**绝不给例题安假的"某年某竞赛/某文献"出处**。有参考资料时忠实转述其事实，不替它编数字或日期。
3. **守用户级写作约束。** 中文；不堆排比；不用感叹号；不说"总的来说"；不用官方陈词滥调与模糊措辞；用比喻要新鲜；浅白易明。

## 工作流

- **第 1 步　摸清主题与素材。** 若用户给了参考资料（PDF/网页/书），先读懂再动笔；以它为事实来源，不臆造。
- **第 2 步　先出骨架，验风格再批量推进。** 先写 `00-目录与体例.md`：书名（仿 "X Made Easy" → "X轻松学"）、四部式结构、每章一个"除恐"主旨、难度标注约定、出处标注规矩。**交用户验风格/深浅，确认对路后再写正文**，避免后面整本跑偏。
- **第 3 步　逐章写，一章一个概念。** 每章严格走下面的"五段式"。讲完就用，用熟再走下一章。
- **第 4 步　写 `99-结语与附录.md`。** 包含：练法/用法、术语小词典、一页纸速查、各章工具速查、出处与难度说明（落实铁律 2）。
- **第 5 步（可选）合成 PDF。** 两条已验证路线（网页版 / LaTeX 版）与全部踩过的坑，见 `references/pdf-build.md`，按那里的命令照做。

## 每章五段式（雷打不动）

1. **开篇除恐**：点破"让你怕的不是难，是它穿的那身西装"，承诺这一章替你脱掉。
2. **白话化**：每个术语先给一个脑子里能看见的画面（比喻/日常类比），再讲规矩。
3. **直觉先行**：先用小孩也懂的办法或小情形切入，再上严格证明/定义。
4. **例子贴身**：两个以上，标难度（☼ 热身 / ☼☼ 正经 / ☼☼☼ 硬骨头）；真题标出处，自编标"自编"；教"怎么想到的"，不只给答案。
5. **收尾**：列"**这一章要带走的东西**"要点，最后单独一句"**就这样。**"

## 风格 DNA 与目录模板

开篇句式、比喻纪律、贯穿例子的用法、难度符号、四部式目录模板，详见 `references/style-guide.md`。写每一章前先扫一眼它。

## 产物落地

- 每本书一个独立文件夹（如 `~/X轻松学/`）。
- 文件命名：`00-目录与体例.md`、`01-章名.md` … `99-结语与附录.md`；**两位数字前缀保证排序正确**（10–16 排在 01–09 之后、99 之前）。
- 贯穿全书用一个真实、贴近用户的例子，跨数章逐步搭起来（如奥数书的"七件工具"、技能书的 `redpen-editor`）。

## 例子

用户说："用《Calculus Made Easy》这种风格写一本教我做 X 的书。"
你应当：读懂 X 的素材 → 出 `00-目录与体例` 骨架交用户验风格 → 逐章按五段式写 → 写结语附录（含出处说明）→ 问是否合成 PDF（按 `references/pdf-build.md`）。
结果：一本结构完整、风格统一、事实可靠、可出 PDF 的"X轻松学"。
