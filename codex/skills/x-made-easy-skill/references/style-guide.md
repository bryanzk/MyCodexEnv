# 风格 DNA 与目录模板

写每一章之前扫一眼这份。源头是 Silvanus P. Thompson《Calculus Made Easy》(1910)。

## 一、汤普森的看家三招

1. **一次只讲一件事。** 一章一个概念/一件工具。讲完立刻用，用熟了再走下一个。绝不在一章里塞两个新概念。
2. **先讲人话，再讲规矩。** 凡术语，先给一个能在脑子里看见的画面，严格定义/证明往后放。
3. **结尾不啰嗦。** 讲明白就收，最后单独一句"就这样。"

## 二、"驱除恐惧"的开篇怎么写

第 1 章永远是"驱除预备的恐惧"：把本主题里最吓人的符号/术语/黑话逐个"脱西装"。模板句式：

- "把多数人挡在门外的，从来不是题难，是它穿了一身吓人的衣服。"
- "这个符号 X，读着唬人，它的全部意思就是【大白话】。"
- 末节专治心病："你以为要天才/要会编程——你不需要。"借汤普森那句改写的猴子谚语收尾：

> 一个笨人能【做成/写明白】的事，另一个笨人照样能。

后续每章的开篇也都先除该章的恐惧，再进入内容。

## 三、术语"脱西装"的标准动作

每个术语三步走：① 给它起个形象的外号（"那个像闪电的 Σ"）；② 一句大白话说穿意思（"就是加起来"）；③ 给一个日常画面或最小例子坐实。例：同余 → "钟面算术，只看余数" → 用"看钟/过 100 天星期几"落地。

## 四、比喻与语言纪律

- 比喻要**新鲜、贴身、能在脑子里看见**（鸽子笼子、三层抽屉、厨房与菜谱、脱西装）。一个好比喻可省一页解释。
- 不堆排比；不用感叹号；不说"总的来说"；不用模糊套话与官方陈词滥调。
- 浅白易明、高雅简洁，"回答如真理一样简单直接"。

## 五、例题规矩

- 每节/每章 ≥ 2 个例子。
- 难度标在例题右上角：☼ 热身（一眼看明思路）/ ☼☼ 正经（要动手，工具在本章）/ ☼☼☼ 硬骨头（章末，啃下很有成就感）。
- **教"怎么想到的"**——展示卡住、试工具、解套的过程，而不是从天而降给答案。可故意演示一次"我中途的犹豫"。
- 真题标出处（经典/民间/经典题型），自编标"自编"；数值全部验算。

## 六、每章收尾固定两件

1. **加粗小标题**"**这一章要带走的东西：**" + 3–5 条要点。
2. 另起一行，单独一句"**就这样。**"

## 七、四部式目录模板（按主题调整部名）

```
# X轻松学
### ——一句副标题（如"一个笨人写给另一个笨人"）
> 仿猴子谚语作题记

## 关于这本书（讲清：本主题的"预备恐惧"是什么，本书怎么除）
## 目录
第一部 · 卸下包袱        —— 除恐 + 把术语脱西装 + 一个统摄性设计
第二部 · 核心工具/概念    —— 本主题真正反复用到的那几件，一件一章
第三部 · 分块上手/实战    —— 把工具放进各子领域，看它们配合
第四部 · 怎么想 / 怎么用  —— 元层面：卡住时的自问清单、如何做得让人服气
尾声：结语 + 附录（术语词典 · 一页纸速查 · 出处与难度说明）
## 难度标注约定
## 出处标注规矩（真题 vs 自编，落实铁律 2）
```

## 八、贯穿例子

全书选一个真实、贴近读者的例子，跨数章从零搭起来——奥数书是"七件工具"逐件登场，技能书是把用户的"B 级编辑红笔报告"做成 `redpen-editor`。让读者读完不只懂概念，手里还多一个真东西。
