# Code Reviewer

## 概述

对最新改动进行结构化审查，优先发现高风险问题，并给出可执行的修复建议。

## 工作流

1. 收集改动
   - 首选 `git diff` 获取变更
   - 聚焦变更文件与相邻代码
2. 分级审查
   - Critical: 安全/数据损坏/稳定性
   - Warning: 质量与可维护性
   - Suggestion: 优化与最佳实践
3. 给出修复建议
   - 指向具体文件与位置
   - 提供替代代码或方案
4. 明确结论
   - Approve / Warning / Block

## 输出要求

- 先列问题，后简述结论
- 按严重性排序，并带文件路径
- 未发现问题需明确说明并提示残留风险
