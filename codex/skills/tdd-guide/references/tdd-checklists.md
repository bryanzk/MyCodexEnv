# TDD 检查清单与示例

## 测试类型要求

- 单元测试（必需）：函数级别行为，覆盖正常/异常路径
- 集成测试（必需）：接口、数据库、外部服务交互
- 端到端测试（关键流程）：覆盖用户主流程

## 必测边界场景

- 空值/零值/空集合
- 非法类型或不合法输入
- 边界值（最小/最大）
- 错误路径（网络失败、超时、依赖不可用）
- 并发或竞态（如适用）
- 大数据量/性能退化（如适用）

## Mock 规范

- 外部依赖必须 mock（数据库、缓存、HTTP、链上 RPC）
- 测试独立，不共享全局状态
- 只 mock 边界，不 mock 被测核心逻辑

## 覆盖率要求

- 默认目标：语句/分支/函数/行 >= 80%
- 关键业务逻辑：尽量达到 100%

## 反模式（避免）

- 测试实现细节而非行为
- 测试依赖顺序或共享状态
- 未运行测试就实现代码

## 示例（伪代码风格）

```text
RED: 编写失败测试 -> 运行确认失败原因
GREEN: 最小实现 -> 运行通过
REFACTOR: 消除重复/优化命名 -> 运行通过
```
