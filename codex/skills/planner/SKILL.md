# Planner

## 概述

提供可执行的实现计划，明确需求与验收标准、影响范围、执行顺序和测试策略，确保计划可落地、可验证。

## 工作流

1. 需求分析
   - 明确目标、范围、输入/输出、非功能约束
   - 列出验收标准与假设
2. 架构/代码梳理
   - 快速定位相关模块与类似实现
   - 识别可复用模式与潜在影响面
3. 任务拆解
   - 每步包含：动作、文件路径、依赖、风险
   - 形成可验证的增量步骤
4. 实施顺序与测试策略
   - 按依赖排序，支持分阶段验证
   - 指定单元/集成/E2E 的覆盖点
5. 风险与缓解
   - 明确潜在回归点与缓解措施

## 输出要求

- 计划必须包含：需求、架构变更点、步骤、测试、风险、成功标准
- 使用明确文件路径与具体动作
- 计划粒度以 1-2 天可完成的步骤为宜
