# Ars Contexta Rethink

## 概述

用于挑战当前假设，避免系统固化。关注“哪些结论可能已过时或偏置”。

## 快速流程

1. 列出当前关键假设。
2. 对每个假设寻找反例与冲突证据。
3. 判定：保留 / 修订 / 废弃。
4. 把变更回写到相关笔记与流程。

## 输出契约

- 被挑战假设列表
- 判定结果与理由
- 需要触发的后续动作（通常是 reweave/verify）

## 常见错误

- 把 rethink 变成纯讨论，不落地更新。
- 没有证据就推翻长期结构。
